Robotics & Physical AI

Current evidence and open directions.

The research map connects recurring questions across projects. Project 01 provides the current quantitative VLA evidence, while Project 02 extends the archive with operational xArm6 teleoperation and demonstration-collection infrastructure.

Projects 01–02

Validated policy evidence and active infrastructure.

Quantitative task results still apply only to the Dobot E6 system. The xArm6 project currently documents teleoperation and collection infrastructure, not a trained VLA policy or transfer result.

01

Current

Efficient VLA Adaptation

We preserve pretrained language knowledge while adapting the robot-facing parts of the policy. The representative v23 configuration starts from π0.5, applies LoRA to SigLIP blocks 18–26, and adapts the action expert.

  • Representative checkpoint: 20,000 steps
  • Current action horizon: 16
  • Local dataset: 198 episodes
02

Current

Real-Time ROS2 Execution and Safety

The runtime bridges policy inference and physical robot commands without using MoveIt2 in the current VLA path. It validates action chunks and enforces command-level safety constraints before execution.

  • WebSocket policy server
  • 20 Hz control loop
  • ServoJ · MovJ · RelJointMovJ
  • Joint-delta, tool-height, and camera checks
03

Current

Robot Demonstration Data and Benchmarking

Dual-camera frames, robot state, suction state, actions, and task prompts are aligned into a LeRobot-compatible dataset for training and evaluation.

  • HIK + ZED RGB
  • 42,495 frames
  • Object, color, and pick-and-place tasks
04

Current

Vision and Action Expert LoRA Analysis

The training repository contains controlled layer-range variants that examine where adaptation capacity is most useful across the SigLIP tower and the action expert.

  • Scoped vision LoRA
  • Action-expert rank and layer studies
  • v23 selected as current representative
05

Current infrastructure · Project 02

Immersive Teleoperation and VLA Demonstration Collection

Quest 3 WebXR control is bridged to an xArm6 + G2 through ROS2, while HIK/ZED observations, robot state, gripper commands, prompts, and anchor events are collected as synchronized episodes.

  • Quest 3 · WebXR
  • xArm6 + G2
  • 16 Hz episode collection

Future research

Cross-Robot and End-Effector Transfer

The second robot infrastructure is now operational, but policy training and transfer evaluation remain future work. A later study can investigate a canonical action interface that separates arm motion from end-effector semantics.

Cross-project research axis

Robot Data & Runtime Infrastructure.

This axis studies how observations, commands, task context, demonstrations, and runtime records remain synchronized and reproducible across both human-operated collection and autonomous policy execution.

Project 01 · Collection & autonomous runtime

E6 coordinates → episodes → policy execution

Coordinate-guided hardware sessions produce aligned demonstrations. The deployed ROS2 runtime then coordinates camera and robot observations, an external WebSocket policy server, supervised action execution, optional context and voice nodes, Foxglove visualization, and MCAP recording.
