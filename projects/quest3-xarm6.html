Stage 2 · Operational collection infrastructure

Quest 3 → xArm6 teleoperation and VLA data pipeline.

An operational ROS2 system that separates real-time Quest controller teleoperation from synchronized demonstration collection using dual cameras, robot state, gripper commands, and a browser-based episode monitor.

System snapshot

Two parallel paths, one hardware owner.

Five ROS2 packages organize eleven running nodes. The manipulation and collection paths remain separate processes while sharing only published state and image topics.

Single-owner boundary

One process owns each physical device.

Only ufactory_driver connects directly to the robot SDK. Only the HIK and ZED camera publisher nodes open their respective camera devices. Every other process consumes existing ROS2 topics; a second direct connection would risk socket or device conflicts.

Current scope

This project page documents an operational teleoperation and data-collection infrastructure. It does not yet claim a trained xArm VLA policy, task-success results, or cross-robot transfer performance.
